# Sprint 8: WordPress + Shopify Publishing

> **Weeks:** 15-16 | **Capacity:** ~40h (20h/week) | **Committed:** 32h
> **Theme:** Two major CMS publishing adapters for the highest-demand platforms

---

## Sprint Goal

Build the WordPress connector plugin (PHP) and Shopify publishing adapter so that articles generated by ChainIQ can be published directly to the two most common platforms used by MENA publishers.

## Key Deliverables

After this sprint, the developer can:

1. **Install chainiq-connector WordPress plugin** -- activates on WordPress 5.8+ / PHP 7.4+
2. **Configure via wp-admin** -- settings page with API key, site connection, publish preferences
3. **Publish articles to WordPress** -- wp_insert_post() with universal builder compatibility (Gutenberg, Elementor, Classic)
4. **Set SEO meta** -- Yoast and RankMath meta fields populated automatically (title, description, focus keyword, schema)
5. **Map categories and tags** -- automatic matching to existing WordPress taxonomies
6. **Upload featured images** -- from CDN URL to WordPress media library
7. **Publish to Shopify** -- Blog Article API creates articles as drafts with metadata
8. **Enrich with products** -- product-aware content inserts relevant product links into articles
9. **Track publish history** -- all publish actions recorded in publish_records with status

---

## Exit Criteria

All must pass before Sprint 8 is marked complete:

- [ ] WordPress plugin activates cleanly on WordPress 5.8+ / PHP 7.4+ (test on fresh WP install)
- [ ] wp-admin settings page renders with API key field, connection test button, and preferences
- [ ] Articles published via wp_insert_post() render correctly in Gutenberg, Elementor, and Classic Editor
- [ ] Yoast SEO meta fields populated: title, meta description, focus keyword
- [ ] RankMath SEO meta fields populated: title, description, focus keyword, schema
- [ ] Category/tag mapping creates new terms if they don't exist, maps to existing if they do
- [ ] Featured image uploaded to WordPress media library and set as post thumbnail
- [ ] Migration 014 creates platform_connections table
- [ ] Migration 015 creates publish_records and image_upload_log tables
- [ ] Shopify adapter creates blog articles via Admin Blog API as drafts
- [ ] Product enrichment adds relevant product links based on article topic
- [ ] Both adapters write publish_records with: article_id, platform, status, remote_url, published_at
- [ ] Error handling: network failure, auth failure, rate limit all produce clear error and retry guidance

## Risk Flags

| Risk | Likelihood | Impact | Mitigation |
| --- | --- | --- | --- |
| WordPress builder compatibility issues (Gutenberg blocks vs. raw HTML) | Medium | High | Use wp_insert_post() with raw HTML -- universal at DB level; test all 3 editors |
| PB-002 exceeds 20h estimate (largest single task) | Medium | High | If >20h by Day 8, defer Shopify (PB-003) to Sprint 9 and extend PB-002 |
| Shopify Blog API rate limiting (2 req/s) | Low | Medium | Backoff on 429, batch operations where possible |
| PHP plugin security vulnerabilities | Medium | High | Validate all input, use nonces, sanitize output, follow WordPress coding standards |
| Yoast/RankMath API changes between versions | Low | Medium | Target current stable versions; use documented meta key names |

- Test accounts for WordPress (local or staging) and Shopify (dev store) preserved for regression testing
