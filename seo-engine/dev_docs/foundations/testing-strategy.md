## 5. Test Data Strategy

| Layer | Strategy | Notes |
|-------|----------|-------|
| Unit tests | In-memory fixtures | No external dependencies |
| Integration tests | Supabase test project | Separate Supabase project for CI |
| E2E tests | Mock server + seeded data | Use `dev_docs/mocks/server.js` |
| Production | No test data | RLS prevents cross-user contamination |

### Mock Server for E2E

The mock API server (Step 10.6) serves as the backend for E2E tests:
```bash
# Start mock server → run Playwright → stop
MOCK_LATENCY=0 node dev_docs/mocks/server.js &
npx playwright test
kill %1
```

## 7. Enforcement Rules

1. **Every new endpoint** must have at least 3 unit tests (happy path, auth failure, validation error)
2. **Every new database table** must have a migration rollback test
3. **Every new dashboard page** must have an E2E smoke test (once Playwright is set up)
4. **Security-critical paths** (auth, encryption, path validation) require ≥90% branch coverage
5. **No PR merges** with failing tests (enforced by CI)
6. **Proof artifacts** required for task completion (console output, screenshots)

## 8. Test Commands

```bash
# Run all bridge tests
npm test

# Run specific suite
node --test tests/auth-middleware.test.js

# Run with verbose output
node --test --test-reporter=spec tests/

# Run smoke tests only (future)
node --test tests/smoke/

# Run integration tests (requires running bridge)
npm run bridge &
node --test tests/integration/

# Run E2E tests (future — requires Playwright + mock server)
cd dashboard && npx playwright test
```
