## Email Design & Technical Specs

### Design Guidelines
- **Template:** Clean, minimal design — text-heavy, not image-heavy
- **Width:** 600px max
- **Font:** System fonts (Arial, Helvetica) for maximum compatibility
- **Brand colors:** Navy (#1a2332) headers, gold (#c9a84c) CTAs, white (#ffffff) background
- **Images:** Max 1-2 per email, always with alt text
- **Arabic emails:** RTL layout, Arabic web font (Noto Sans Arabic or Tajawal)
- **Mobile:** All emails must render correctly on mobile (50%+ opens will be mobile)

### Sender Configuration
- **From name:** "Khalid from ChainIQ" (personal name + brand)
- **From email:** khalid@chainiq.io
- **Reply-to:** khalid@chainiq.io (goes to real inbox)
- **Physical address:** Required by CAN-SPAM — use agency address

### Deliverability Checklist
- [ ] SPF record configured for chainiq.io
- [ ] DKIM signing enabled
- [ ] DMARC policy set (start with p=none, move to p=quarantine after 30 days)
- [ ] Dedicated sending domain or subdomain (mail.chainiq.io)
- [ ] Warm up sending domain: start with 50 emails/day, increase 25%/week
- [ ] Unsubscribe link in every email (legally required)
- [ ] Plain text version of every email
- [ ] Test rendering in Gmail, Outlook, Apple Mail, Yahoo before sending

### A/B Testing Plan
- Test subject lines on every email (min 2 variants)
- Test CTA button text on welcome sequence
- Test send times: 9 AM GST vs 2 PM GST vs 7 PM GST
- Minimum sample size: 100 recipients per variant before declaring winner

### Key Metrics to Track
| Metric | Benchmark | Action if below |
|--------|-----------|-----------------|
| Open rate | >25% | Improve subject lines, check deliverability |
| Click rate | >3% | Improve CTA copy, email design |
| Unsubscribe rate | <0.5% per email | Reduce frequency, improve relevance |
| Welcome → Trial activation | >60% | Improve onboarding emails |
| Nurture → Conversion | >5% | Improve value proposition, offer |
| Win-back → Reactivation | >8% | Improve offer, address churn reasons |
