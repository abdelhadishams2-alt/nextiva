## Detailed Findings

### 1. Synchronous I/O (Medium)

**Rule:** Never use `readFileSync`/`writeFileSync` in request handlers.

| File | Line | Pattern | Impact |
|------|------|---------|--------|
| `bridge/server.js:1449` | `fs.writeFileSync(pidFile, ...)` | PID file write at startup | **Low** — only runs once at boot, not in request path |
| `bridge/supabase-client.js:37` | `fs.readFileSync(configPath, ...)` | Config file read | **Medium** — runs on config cache miss, blocks event loop briefly. Mitigated by mtime-based caching. |

**Verdict:** Both are acceptable for now. The supabase-client sync read happens rarely (config changes) and is cached. The PID write is startup-only. Neither blocks request handling under normal operation.

**Action:** Monitor. Convert to async in Phase 5 if config reads become more frequent.

### 2. Hardcoded Supabase Anon Key (Low — By Design)

| File | Line | Key Type |
|------|------|----------|
| `bridge/supabase-client.js:18` | Anon key fallback | Public (RLS-protected) |
| `config/.supabase.json:3` | Anon key config | Public (RLS-protected) |

**Verdict:** This is the Supabase **anon** key, which is designed to be public. Supabase's architecture requires the anon key on the client side — RLS policies protect data. The `service_role` key (which bypasses RLS) is correctly stored as an environment variable only.

**Action:** None. This is correct behavior per Supabase's security model.

### 3. CORS Wildcard (Medium)

| File | Line | Pattern |
|------|------|---------|
| `bridge/server.js:244` | `Access-Control-Allow-Origin: *` | Global CORS handler |
| `bridge/server.js:1083` | `Access-Control-Allow-Origin: *` | SSE endpoint |

**Verdict:** Acceptable for localhost-only development. **Must be restricted for production deployment** — replace `*` with the dashboard's origin (`https://dashboard.chainiq.io`).

**Action:** Track in DR plan. Production deployment checklist (Step 17) must enforce CORS origin restriction.

### 4. Console.log in Server (Low)

| File | Lines | Pattern |
|------|-------|---------|
| `bridge/server.js:1451-1455` | Startup banner | 5 console.log calls |

**Verdict:** Startup banner only. Not in request handlers. Structured logger (`bridge/logger.js`) handles all request-path logging.

**Action:** None.
