## Minor Finding (Non-Blocking)

**SSE auto-close gap:** `subscribeToProgress()` in `api.ts` line 290 auto-closes EventSource on `"completed"`, `"failed"`, and `"cancelled"` — but NOT on `"complete"` (without 'd'). The page handles `"complete"` by setting genState to `"complete"`, but the EventSource stays open until `handleReset` is called (user clicks "Generate Another") or the page unmounts. This is a minor resource leak — the SSE connection lingers in the "complete" state until user interaction.

**Recommendation:** Add `"complete"` to the auto-close list in `subscribeToProgress()`, or call `unsub()` immediately when genState transitions to complete. Does not block PASS — the connection closes eventually and causes no functional harm.

## Patterns Established

1. **Discriminated union for UI state machine** — `GenerationState` union type with `phase` as the discriminant; `Extract<GenerationState, { phase: "running" }>` for prop typing. Use this pattern for any multi-phase async UI.

2. **SSE with token-as-query-param** — `EventSource` does not support Authorization headers; pass token as `?token=` query param. Bridge server must accept this for SSE endpoints.

3. **Settings-seeded form defaults** — mount effect fetches user settings and seeds all form fields before first render. Quota is fetched in parallel. Pattern: `Promise.all([fetchSettings(), fetchQuota()])`.

4. **useRef for cleanup handles** — `unsubRef.current` stores the SSE unsubscribe function; cleanup effect calls it on unmount. Prevents memory leaks across navigation.

5. **Quota-gated framework list** — FRAMEWORKS array is filtered against `quota.frameworks.allowed` when quota data is present. Declarative plan enforcement in the UI layer.

6. **Inline QuotaIndicator** — Quota as a card component above the form, with color-coded progress bar and badge. Pattern for any usage-gated feature.

7. **Skeleton-first loading** — Full-page skeleton matching form layout; no flicker of empty form before defaults load.
