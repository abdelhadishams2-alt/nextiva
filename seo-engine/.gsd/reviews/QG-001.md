## Patterns Established

1. **Quality engine modules in `engine/` directory** -- domain logic separated from bridge server
2. **Checklist definition pattern** -- array of `{ id, category, label }` with `check(id, status, value, expected, message)` evaluator
3. **E-E-A-T scoring pattern** -- dimension array with `{ name, score, maxScore, description }` and letter grade thresholds
4. **Suggestion map pattern** -- keyed by item ID with `{ suggestion, impact }` templates
5. **Article retrieval pattern for quality endpoints** -- `supabase.getArticle(token, id)` with fallback fields (`html_content || content`, `keyword || topic`)
